Project mission

Cosmetic safety is a critical issue that disproportionately affects people of color and low-income communities, who often lack access to data-driven skincare guidance. This tool democratizes ingredient transparency by combining a government-backed safety database with personalized skin-type scoring — making it easy for anyone to understand what's actually in their products.

Key features

Real government data

Parses the full California Safe Cosmetics Program CSV — the only state-mandated registry of harmful cosmetic chemicals in the US.

Skin-type scoring

Scores each ingredient using a weighted formula: 60% safety + 40% effectiveness, adjusted by oily, dry, sensitive, or combination profiles.

Three formulations

After analysis, the app suggests a safest option, a moderate option, and a list of ingredients to avoid — tailored to your profile.

Software design

Data layer

CSVParser reads 114k rows, handles quoted commas, BOM encoding, and malformed rows with try/catch

Logic layer

SkinScorer applies the Score<T> template to rate safety (60%) + effectiveness (40%) per skin profile

UI layer

MainWindow renders sliders, a scored results table, Qt Charts bar chart, and formulation suggestions

Files implemented

csvparser.h / .cpp

Parses the CSCP CSV into IngredientRecord structs. Handles quoted fields with commas, skips malformed rows, and uses try/catch exception handling. Implements searchByChemical(), searchByProduct(), and searchByBrand().

ingredient.h / .cpp

Defines the Ingredient base class with two subclasses: ActiveIngredient (beneficial ingredients like niacinamide) and AllergenIngredient (CSCP-flagged chemicals, score penalized by 50%). The static fromRecord() factory method converts any CSV row into the correct subclass.

Score.h

Generic Score<T> template class — satisfies the honors C++ template requirement. Works for int, double, or float score values. Includes getColor() (green/orange/red) and getSafetyLabel() for UI display.

SkinScorer.h / .cpp

Scoring engine that takes a SkinProfile (skin type + three slider values) and scores all ingredients. Safety score starts at 8, penalizes CSCP allergens by −4 and discontinued products by −2. Effectiveness score adjusts ±3 based on skin-type keyword lists and ±2 from concern sliders.

mainwindow.h / .cpp

Qt GUI with skin profile group box (dropdown + 3 sliders with live labels), search panel, scored results table with color-coded cells, Qt Charts bar chart of top 10 ingredients, and three formulation suggestion labels. Connects signals to onSearchClicked(), onAnalyzeClicked(), and onSkinTypeChanged() slots.

Key code — Score<T> template

// Score.h — generic template, satisfies honors requirement template <typename T> class Score { public: Score(T val, T maxVal, QString lbl) : value(val), maxValue(maxVal), label(lbl) {} QString getColor() const { double pct = getPercentage(); if (pct >= 0.7) return "#4CAF50"; // green — safe if (pct >= 0.4) return "#FF9800"; // orange — moderate return "#F44336"; // red — danger } private: T value; T maxValue; QString label; };

Challenges

  • Qt 6 removed setCodec() — replaced with setEncoding(QStringConverter::Utf8)
  • CSV file path not found at runtime due to macOS .app bundle structure
  • QVector::contains() requires == operator — solved using QSet<int> ID deduplication
  • Linker errors from missing function definitions between .h and .cpp files
  • std::min type mismatch between int and qsizetype — fixed with qMin()

Data source

California Safe Cosmetics Program (CSCP) Database — California is the only state that requires companies to report harmful cosmetic ingredients. The dataset contains 114,635 records of products containing known carcinogens or reproductive toxins, maintained by the California Department of Public Health.

California Open Data — Chemicals in Cosmetics

Scoring formula

overall = (safetyScore × 0.6) + (effectivenessScore × 0.4) safetyScore starts at 8 −4 if CSCP allergen keyword matched −2 if product discontinued clamped to [1, 10] effectivenessScore starts at 5 +3 if beneficial for skin type −3 if ingredient to avoid for skin type +2 slider boost (acne / dryness / hyperpig) clamped to [1, 10]
